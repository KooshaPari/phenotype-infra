### Problem

The old `ci.yml` bundled **fmt, check, and clippy into one job** (`check`).
This means:
- Branch protection rules could not distinguish between a **formatting failure**
  vs a **compilation failure** vs a **clippy lint failure**.
- All three were a single check mark. To enforce them individually the
  jobs must be split into independently nameable checks.

## Tier-0 Enforcement Contract

Every PR targeting `main` must pass these required checks before merging:

| Gate | Job ID | What it verifies |
|------|--------|-----------------|
| Build | `build` | `cargo check --workspace --all-targets --locked` |
| Unit Tests | `test-unit` | `cargo test --workspace --locked --all-features` |
| Format + Lint | `fmt-clippy` | `cargo fmt --check` + `cargo clippy -- -D warnings` |
| Security Audit | `lint` | `cargo deny` + `cargo audit` |

These four gates form the **Tier-0** surface. All must be green before a PR
can be merged into `main`.

## Branch Protection Configuration

In addition to the workflow changes, the **main branch protection rule** in
the GitHub repo settings must have these checks set as **"Require status
checks to pass before merging"**:

- `build`
- `test-unit`
- `fmt-clippy`
- `lint`

These correspond 1:1 to the job `name:` values in `ci.yml`.
