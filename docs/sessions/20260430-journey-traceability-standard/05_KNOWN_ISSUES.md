# Known Issues

## Current Gaps

- Existing repos may not yet have the keyframe assets captured.
- Some docs pages may still rely on prose only.
- Not every repo has a docs-site renderer that can embed the same components
  without a small adaptation.

## Workaround

Until a repo is fully adopted, the standard should be referenced explicitly in
the docs page and the missing capture should be called out as a blocker.

## Follow-Up

Audit the major repos and mark which flows already have:

- keyframes
- recordings
- a stable tape id
- a repo-local docs page
