# DAG / WBS

## Dependency Order

1. Define the standard in governance.
2. Link the standard from the shared parity doc.
3. Roll the standard into repo-level docs indices.
4. Capture missing journeys where needed.

## Critical Path

The governance doc is the blocker for everything else. Once that exists, each
repo can adopt the same contract without inventing its own pattern.

## Next Targets

The first adoption pass should focus on the docs hubs and registries that can
fan the standard out to many repos:

1. `phenodocs` - the documentation federation hub.
2. `PhenoHandbook` - the cross-repo patterns registry.
3. `PhenoProject` - the workspace-level docs/worklog hub.
4. Active product repos with existing docs sites.
