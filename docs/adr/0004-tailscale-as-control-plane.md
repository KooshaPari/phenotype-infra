# ADR 0004 — Tailscale as Control Plane

- **Status:** Accepted
- **Date:** 2026-04-24

## Context

The 7-node mesh spans 4 cloud providers plus a home machine. We need:

- A single overlay network with stable names across dynamic public IPs.
- ACL-driven access control between nodes.
- Zero-config NAT traversal (home Mac is behind a residential router).
- Ephemeral node support for burst runners.
- DNS that works without running BIND.

## Decision

Use **Tailscale** (free tier — up to 100 devices, 3 users) as the mesh control plane. Every node joins the same tailnet. MagicDNS gives us stable names:

- `oci-primary`, `oci-secondary` — cloud backbone
- `gcp-e2` — tertiary runner
- `home-mac` — heavy runner
- `hetzner-burst` — Phase-2 spillover

**ACLs** enforce role-based segmentation: Woodpecker agents can reach the server, but not each other; the home Mac cannot reach cloud nodes except the Woodpecker server; admin SSH is gated to a specific tag.

**Ephemeral auth keys** (pre-authorized, 1-hour TTL, auto-expiring) are generated by Terraform for burst/Phase-2 nodes.

## Consequences

**Positive**

- Zero public ingress on 5 of 7 nodes (only Cloudflare Tunnel exits are public).
- MagicDNS eliminates the "what's the IP again" failure mode.
- ACLs are a single JSON file in the Tailscale admin console — auditable and revertible.
- NAT traversal is automatic; residential home Mac joins without port forwarding.

**Negative**

- Single point of trust: if Tailscale control-plane is compromised, all mesh connectivity is at risk. Mitigated by: (a) Tailscale's open-source client + audited control plane, (b) emergency fallback documented in `docs/specs/rollback-kill-switch-spec.md` (WireGuard raw configs in Vaultwarden).
- Free tier limits (100 devices, 3 users) are ample now but require monitoring.
- Some enterprise features (subnet routers, exit nodes) may trigger paid tier in Phase 2.

## Alternatives considered

1. **Raw WireGuard.** Rejected: manual key rotation, no MagicDNS, no NAT traversal.
2. **Nebula (Slack's overlay).** Rejected: cert rotation complexity; smaller ecosystem.
3. **ZeroTier.** Rejected: slower NAT traversal in practice; less mature ACL model.
4. **Cloud-provider VPN peering.** Rejected: doesn't reach home Mac; per-provider complexity.
